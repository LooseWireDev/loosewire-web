Linux users get left behind by most meeting tools. Otter.ai has no Linux app. Fireflies runs as a browser bot and doesn't need one — but your audio still ends up on their servers. tl;dv requires a Chrome extension that has limited Linux support.

Harknotes is built to run natively on Linux. Here's what that looks like in practice.

## The Linux audio challenge

Meeting transcription on Linux is harder than on macOS for a technical reason: audio routing.

On macOS, the operating system provides a unified audio session API. Capturing both system audio and microphone is straightforward.

On Linux, audio is handled by PulseAudio or PipeWire, depending on your distribution. Both support loopback recording — capturing system audio output — but the configuration differs, and some distros require additional setup.

<Callout type="tip" title="Automatic audio detection">
Harknotes detects whether you're using PulseAudio or PipeWire and configures audio capture automatically. No manual setup required.
</Callout>

## Supported distributions

Harknotes is tested on:

- **Ubuntu 22.04+** — PulseAudio and PipeWire (22.10+)
- **Fedora 39+** — PipeWire (default since Fedora 34)
- **Debian 12** — PulseAudio
- **Linux Mint 21** — PulseAudio
- **Pop!_OS 22.04** — PulseAudio
- **openSUSE Tumbleweed** — PipeWire

DEB and RPM packages are available, covering the major package formats across these distributions — and Harknotes is [open source](https://github.com/LooseWireDev/harknotes), so you can also build it from source on anything else.

<SectionHeading label="Getting set up" />

## Installation

**DEB** — for Debian, Ubuntu, Mint, Pop!_OS:
```bash
sudo dpkg -i harknotes_*.deb
```

**RPM** — for Fedora, RHEL, openSUSE:
```bash
sudo rpm -i harknotes-*.rpm
```

On first launch, Harknotes downloads the Whisper model you pick — from ~75MB for `tiny` up to the large models — sized to your hardware. After that, transcription is fully offline.

## Speaker labels on Linux

Harknotes captures your microphone and system audio as separate streams. Your voice (microphone) is labelled "User"; everyone else comes through system audio and is labelled "Meeting". You can rename speakers on the transcript after the meeting, and full per-person diarisation is on the roadmap.

No cloud processing, no data leaving your machine.

## Why not just use Whisper directly?

You can run whisper.cpp on Linux from the command line. If you're comfortable with that workflow, it works well.
